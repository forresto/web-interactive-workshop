  </body>

CSS

CSS rules can refer to 3 things: tag name, id, or class.

Tag names

Like body, h1, h2, h3, p, span, a, img

#id

Only one element can have a particular id.

.class

Many elements can have the same class. Each element can have more than one class.

All together now

You can refer to an element with any combination of tag name, id, and class.

Web fonts

Typekit costs, Google Web Fonts are free.

In your page's head:

<link href='http://fonts.googleapis.com/css?family=Sofia' rel='stylesheet' type='text/css'>

In the css:

.specialfont{
  font-family: 'Sofia', cursive;
}

Now everything with the class "specialfont" will be in Sophia.

JavaScript + JQuery

It is always good to get the page working and looking good without JavaScript first, and then use it just as much as needed.

JQuery makes it easy to select and change elements on the page.

$()

$("") will match all elements in the same way that CSS rules work:

  • tag name: $("a") gets all links
  • id: $("#maintitle") gets the one element that has id="maintitle"
  • class: $(".classname") gets all elements that have class="classname"

click()

hide(), show(), toggle()

toggle("slow") takes 600ms, toggle("fast") takes 200ms, toggle(2500) takes 2.5s

text()

addclass() and removeClass()

Processing.js

If you want to embed your (Java) Processing sketches directly using Processing.js:


<!DOCTYPE html>
  <html>
  <head>
    <title>Hello Web - Processing.js Test</title>
    <script src="processing-1.4.1.min.js"></script>
  </head>
  <body>
    <h1>Processing.js Test</h1>
    <p>This is my first Processing.js web-based sketch:</p>
    <canvas data-processing-sources="hello-web.pde"></canvas>
  </body>
</html>

Just use <canvas>

Drawing with pure JavaScript to the canvas isn't much more complicated than Processing.
